1. プロダクト概要


目的

地図上で周辺のスポットを検索・探索できるWebアプリケーションを開発する。ユーザーが地図を操作しながら、特定地点周辺のスポット情報を直感的に把握できる体験を提供する。

提供価値

- 地図とリストを連動させ、視覚的にスポットを探索できる
- 「今どこを見ているか」を常に把握でき、ストレスのない操作感を実現する
- 半径を動的に変更し、目的に応じた範囲でスポットを絞り込める

対象データ

全国の観光スポット約500件（name / category / lat / long / address を持つCSV）。データは全国に分散しているため、広域での検索を前提とする。
---

2. 業務要件

- DB上のスポットを地図上にマーカーとして表示する
- ユーザーが指定した中心点・半径に基づき、周辺スポットを抽出する 
	- 該当スポットが存在しない場合は「付近に該当するスポットが存在しません」と表示する
- スポットのカテゴリが視覚的にわかるマーカーを地図上に配置する
- 地図の中心地点の住所を、地図移動に追従してリアルタイム／高頻度で表示する 
	- 目的：ユーザーが「今どこを見ているか」をストレスなく把握できるようにするため
- 初期状態（docker-compose up 直後）からアプリケーションが動作可能である
---

3. 機能要件


① データインポート

- 提供シードデータ（CSV）を docker-compose up 実行時にDBへ自動インポートする
- 初期状態からアプリケーションが動作可能な状態で提出する

② 周辺検索・表示機能

- 地図連携：Google Maps API を用いてスポットをマーカー表示する
	- マーカーはカテゴリごとに色分けする
- リスト連動：地図の表示内容と連動するスポット一覧（リスト形式）を表示する
	- カテゴリでスポットを分類して表示する
- 半径検索：指定中心点から「半径N km以内」のスポットを抽出する
	- 検索半径はUI上で動的に変更可能とする
	- 初期中心点：東京駅（周辺に複数スポットが存在する地点）
	- 初期半径：50km前後（全国分散データのため広域を初期値とする）
- 検索結果ゼロ件時のハンドリング：半径内にスポットが存在しない場合、「付近に該当するスポットが存在しません」と表示する
- 検索呼び出しの最適化：中心座標・半径が変化したときのみ検索APIを呼び出す

③ 地図中心地点の住所表示

- 地図移動中、常に地図中心地点の住所をリアルタイム／高頻度で表示する
- 住所特定には逆ジオコーディングAPI（Google Maps API）を利用する
- ユーザーが「今どこを見ているか」をストレスなく把握できるように実装する
- ローディング表示：地図移動中・住所取得中はスピナーを表示する
- API呼び出しの抑制：
	- 座標のわずかな移動ではAPIを呼ばない（デバウンス）
	- 一度取得した座標→住所の結果をキャッシュし、同一・近接座標の再問い合わせを削減する
---

初期値の設計根拠

提供データは北海道から沖縄まで全国に分散しており、観光地同士の距離が大きい。一般的な店舗検索の数km初期値では大半の地点で0件となるため、初期表示で検索機能を体感できるよう、中心を東京駅・半径を50kmに設定した。
